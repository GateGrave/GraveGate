# GateGrave Documentation Index

Use these documents first.

## Canonical Documents

- `CONSTITUTION.md`
  - repo-wide rules, source-of-truth rules, refusal rules, and authoring discipline
- `PROJECT_ROADMAP.md`
  - high-level project direction and priority order
- `IMPLEMENTATION_STATUS.md`
  - current repo state and what is actually implemented enough to build on
- `MAP_ROADMAP.md`
  - current map-lane priorities and near-term map work
- `MAP_AUTHORING.md`
  - canonical map authoring and compile-to-data workflow

## Supporting Documents

- `closed-alpha-readiness.md`

## Historical Documents

- `history/`
  - dated handoffs
  - repo-safe historical notes
  - project snapshots
  - older status files

Historical documents are context, not policy.

Local-only backups and scratch notes belong outside versioned docs, for example in gitignored backup locations.
